# macOS 相似视频查找器设计规格

## 目标

开发一款可双击运行的原生 macOS 应用。用户选择一个文件夹后，应用结合文件信息和视频画面检测相似视频，将结果分组展示，并允许用户预览后手动决定如何删除单个视频。

第一版最低支持 macOS 14，不包含应用签名、公证或 DMG 安装包。

## 核心流程

1. 用户选择需要扫描的文件夹，并允许应用遍历子文件夹。
2. 应用扫描支持的视频文件，读取文件名、大小、时长、分辨率等元数据。
3. 应用先快速筛选候选视频，再抽取画面进行内容相似度确认。
4. 相似结果按组显示。每个相似组至少包含两个视频。
5. 用户选择视频，在应用内预览或使用系统默认播放器打开。
6. 用户点击删除后，在确认弹窗中选择移到废纸篓、永久删除或取消。
7. 删除成功后立即刷新结果；只剩一个视频的组自动移除。

## 界面设计

采用原生 SwiftUI 三栏检查器布局。

### 左栏：扫描与相似组

- 提供“选择文件夹”和“开始扫描”操作。
- 扫描中显示当前阶段、当前文件、整体进度、已发现视频数量，并提供取消按钮。
- 扫描完成后列出相似组，显示组内视频数量和最高相似度。
- 提供相似度阈值设置，默认仅显示高相似结果。

### 中栏：组内对比

- 展示当前组内所有视频的缩略图。
- 每个视频显示文件名、文件大小、时长、分辨率和完整路径。
- 显示综合相似度及主要判断依据，例如画面接近、时长接近或内容哈希一致。
- 单击选择视频并在右栏查看；不提供自动删除或自动勾选删除。

### 右栏：预览与操作

- 使用系统视频播放器组件播放、暂停和拖动进度。
- 提供“使用默认播放器打开”和“在 Finder 中显示”。
- 提供明显但非默认聚焦的删除按钮。
- 未选择视频时显示操作提示，不显示无效播放器。

## 相似度检测

检测分为快速筛选和画面确认两个阶段。

### 快速筛选

- 递归发现常见视频格式。
- 读取文件名、文件大小、时长和分辨率。
- 对大小完全相同或高度接近的候选计算内容哈希；哈希一致时判定为完全重复文件。
- 使用文件名标准化、时长差、大小比例和分辨率关系构建候选集合，避免对所有视频进行完整的两两画面对比。

### 画面确认

- 使用 AVFoundation 在视频的多个相对时间点抽帧，至少覆盖开头、中间和结尾区域。
- 使用 Vision 生成图像特征并比较对应及邻近采样画面。
- 无法在某个时间点抽帧时跳过该帧；有效画面不足时降低置信度，不把结果伪装成高相似。
- 综合画面相似度、时长、大小、文件名和内容哈希计算最终分数。
- 使用图连通分量构建相似组，使 A 与 B、B 与 C 达到阈值时可以形成同一组；界面仍展示视频之间的具体分数，避免隐藏弱关联。

第一版结果仅保存在内存中，不维护长期索引数据库。

## 删除安全策略

- 每次删除都必须弹出确认窗口，并显示目标文件名和完整路径。
- 确认窗口提供“移到废纸篓”“永久删除”和“取消”三个明确选项。
- “取消”为默认安全操作。
- 永久删除选项必须包含不可恢复警告，并需要第二次确认。
- 删除失败时保留当前结果并显示具体错误，不提前从界面移除文件。
- 成功删除后更新相似组；组内不足两个文件时移除整个组。

## 技术架构

- **UI：** SwiftUI，使用 NavigationSplitView 构建三栏界面。
- **视频处理：** AVFoundation 读取元数据、生成缩略图、抽帧和播放。
- **画面特征：** Vision 图像特征打印及距离比较。
- **文件操作：** FileManager、NSWorkspace 和 macOS 文件选择面板。
- **并发：** Swift Concurrency；扫描和画面分析在非主线程执行，通过可取消任务报告进度。

主要模块：

- `VideoScanner`：发现文件并读取元数据。
- `CandidateMatcher`：执行快速筛选和内容哈希判断。
- `FrameFeatureExtractor`：抽帧并生成 Vision 特征。
- `SimilarityEngine`：合并各项分数并生成相似关系。
- `SimilarityGrouper`：把相似关系转换为可展示的分组。
- `DeletionService`：执行废纸篓和永久删除操作。
- `ScanViewModel`：管理扫描状态、取消、选择和删除后的结果更新。

## 异常处理

- 无权限读取的文件、损坏视频及不支持格式不会中断整个扫描。
- 扫描完成后提供问题摘要，列出跳过数量和原因。
- 文件在扫描后被外部移动或删除时，预览和删除操作显示文件已不存在，并刷新对应分组。
- 用户取消扫描时停止创建新任务，丢弃本次未完成结果并显示“扫描已取消”；第一版不展示部分相似结果。
- 大文件夹扫描过程中保持界面可响应，并限制并发抽帧数量，避免内存和 CPU 突增。

## 测试策略

- 单元测试文件名标准化、元数据评分、综合评分和阈值判断。
- 单元测试相似关系分组，包括链式关系、孤立视频和删除后组更新。
- 使用依赖注入测试扫描取消、错误汇总、删除成功和删除失败状态。
- 使用小型本地视频夹具验证 AVFoundation 抽帧和 Vision 特征流程。
- UI 测试覆盖选择文件夹后的状态、组切换、视频选择及删除确认窗口。
- 手动验收真实视频播放、Finder 打开、移到废纸篓及永久删除二次确认。

## 验收标准

- 应用可构建为可双击运行的 `.app`。
- 用户可选择文件夹并取消正在进行的扫描。
- 内容相同但文件名、分辨率或压缩方式不同的视频能够进入候选检测并在画面相似时成组。
- 用户可以在应用内播放选中的视频，也可以使用默认播放器打开。
- 应用绝不自动删除视频；每次删除均由用户明确选择删除方式。
- 删除失败不会造成界面误报；删除成功后分组立即正确更新。
- 扫描损坏或不可读视频时应用不崩溃，并能显示问题摘要。

## 非目标

- 云端分析、账号系统和跨设备同步。
- 自动删除、自动选择“较差版本”或批量无人值守清理。
- 长期媒体索引和后台持续监控。
- App Store 发布、开发者签名、公证和 DMG 分发。
